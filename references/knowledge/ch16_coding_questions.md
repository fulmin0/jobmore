# Ch16: Coding Questions

## Summary
Brief summary — this chapter is relevant primarily for technical PM roles and engineering-to-PM transitions. Most PM interviews do not include coding questions; this chapter is a long reference for the minority of roles (and companies like Google, Microsoft) that do test coding.

## Key Concepts
- **When coding questions appear**: Technical PM roles, APM programs, companies with strong engineering cultures (Google, Microsoft, Amazon in some cases), roles explicitly labeled "Technical PM" or "TPM"
- **What's tested** (PM-specific level, not SWE level):
  - Basic data structures: arrays, linked lists, hash maps, trees
  - Algorithm complexity: Big-O notation, why a solution is O(n) vs O(n²)
  - System design concepts: databases, APIs, caching, queues — more common than actual coding
  - SQL: basic queries, joins, aggregations — very commonly tested for data-fluent PM roles
- **What's NOT typically tested**: Competitive programming, complex dynamic programming, advanced algorithms — that's SWE territory
- **SQL relevance**: This is the most practically important coding skill for PMs. Basic `SELECT`, `JOIN`, `GROUP BY`, `HAVING`, `WHERE` fluency is expected in data-heavy PM roles

## Direct Quotes / Signals Worth Preserving
- The chapter itself notes: "Most PM interviews don't include coding questions. If you're not applying to technical PM roles or engineering-heavy companies, you can skip this chapter."

## Relevance to Job Search Pipeline
- **SQL prep**: For any role with "data-driven" or "analytics" in the JD, prepare 2–3 SQL query questions (funnel analysis, cohort retention, event aggregation)
- **System design overlap**: Technical PM system design questions (Ch14 system_design_prep.md) draw from the architecture concepts introduced here — scale, latency, tradeoffs
- **De-prioritize**: Unless targeting Google APM, technical PM, or a role that explicitly mentions coding — skip deep study here and invest time in Ch12, Ch14, and Ch17
